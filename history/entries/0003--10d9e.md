## Challenge change

The challenge is now a **boolean gate bootstrap** (`REQUIRED_MESSAGE_BITS = 2`: 1 padding +
1 message bit, 2 messages) — the rs-tfhe / CGGI gate setting, apples-to-apples with rs-tfhe.
Entries 0000–0002 were the earlier **4-message** spec (`message_bits = 3`) and are not directly
comparable (harder noise budget). The boolean gap (q/4) is roomier, so the LWE dimension drops to
rs-tfhe's **n=700**.

## Result — beats rs-tfhe head-to-head

Same machine (Apple M3 Pro), same boolean params (n=700, N=1024, ℓ=3), measured back-to-back:

| | one bootstrap |
|---|---|
| **this solution** | **~14.3–14.5 ms** (best) |
| rs-tfhe `gate_nand` (rustfft) | 14.60–14.80 ms |

The win is attributable to our **hand-written radix-4 split-format SIMD FFT** (forward/inverse
~1.6 µs each, vs rs-tfhe's rustfft ~1.9 µs) plus the fully-SIMD FFT scalar passes — same scheme,
same dimensions, faster transform. rs-tfhe's SPQLIOS AVX path is disabled; it runs on rustfft.
