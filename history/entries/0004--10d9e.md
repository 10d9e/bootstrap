## The idea (mathematical)

128-bit security fixes only the **module-LWE lattice dimension** `D = k·N`, not `N` itself. But
the blind-rotation FFT work is **superlinear in N**:

```
FFT work per external product  ∝  (k+1)·(N/2)·log(N/2)  =  (k+1)/(2k) · D · log(D/2k)
```

The factor `(k+1)/(2k)` **decreases as k grows**, so splitting the GLWE as a higher-rank,
shorter-ring module trades one big transform for several much cheaper ones at identical
security and key-switch. Sweeping  (all with ):

| (k, N) | best ms | note |
|--------|---------|------|
| 1, 1024 | 14.4 | (entry 0003) |
| 2, 512  | 11.7 | |
| **4, 256** | **9.9** | **sweet spot** |
| 8, 128  | 11.5 | regresses — bsk size ∝ (k+1)² and per-transform overhead overtake the FFT savings |

The optimum is interior: FFT savings ∝ (k+1)/(2k) keep helping, but the bootstrap-key size and
pointwise-MAC count grow ∝ (k+1)², and the memory-bound key stream + per-transform overhead win
at large k. **k=4 (N=256)** is best.
